   Что такое Git?
Это самая популярная распределенная система управления версиями, используемая разработчиками для отслеживания изменений в исходном коде, совместной работы над проектами и управления версиями файлов. Она позволяет фиксировать (коммитить) каждое изменение, создавать ветки для параллельной работы и откатываться к старым версиям.
   Что такое репозиторий?
Это цифровое хранилище, где хранятся, обновляются и поддерживаются данные, чаще всего исходный код программного обеспечения, файлы проектов, документация или пакеты программ
   Зачем нужны коммиты?
Для фиксации изменений (создания «снимков») проекта в определенный момент времени, обеспечивая контроль версий. Они позволяют сохранять историю разработки, откатываться к рабочим версиям при ошибках, параллельно работать над разными задачами в ветках и эффективно взаимодействовать в команде.

   ```
 Что такое Pull Request?
 Pull Request (в переводе — «запрос на вытягивание изменений») — это механизм, который используется при командной разработке для контроля качества кода. Разработчик создаёт отдельную ветку, вносит в неё изменения, а затем с помощью Pull Request предлагает эти изменения объединить с основной веткой проекта (обычно main или develop).
Проще говоря: Pull Request — это «заявка» на добавление вашего кода в общий проект, которую другие разработчики должны проверить и утвердить.

 Зачем нужны ветки?
Ветки (branches) нужны для изоляции изменений. Эксперименты: Можно тестировать новые идеи, не боясь «сломать» основной рабочий код. Параллельная работа: Несколько человек могут одновременно работать над разными задачами (например, один чинит баг, другой делает новую фичу), не мешая друг другу. Безопасность: Вы объединяете ветку с основной только тогда, когда код проверен и готов.
