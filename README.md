## 1. Определение репозитория
**Репозиторий** — это цифровое хранилище для управления проектом. В нем хранятся:
* Исходный код программного обеспечения;
* История всех изменений (версии);
* Сопутствующая документация и файлы настроек.

## 2. Работа с коммитами в GitHub Desktop
Процесс фиксации изменений (Commit) состоит из следующих шагов:
1. **Внесение изменений:** Редактирование файлов проекта.
2. **Оформление:** В левом нижнем углу интерфейса GitHub Desktop необходимо заполнить заголовок (Summary) и описание.
3. **Фиксация:** Нажатие кнопки «Commit to main» для сохранения локальной версии.

## 3. Синхронизация с удаленным сервером
Для отправки локальных изменений на сервер GitHub используется команда **Push**:
* После создания коммита данные отправляются в облако нажатием кнопки «Push origin».
* Это позволяет синхронизировать локальный проект с удаленным репозиторием.

## 4. Роль веток в разработке
Ветки позволяют вести разработку **изолированно**. Это необходимо для:
* Создания новых функций без риска испортить основной рабочий код (`main`).
* Параллельной работы нескольких участников команды над разными задачами.
* Тестирования экспериментальных идей.
